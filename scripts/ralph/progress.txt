## Codebase Patterns
- Monorepo uses npm workspaces with packages/* and apps/* layout
- Root tsconfig.json is the base; each package extends it with its own rootDir/outDir
- packages/core uses ESM (`"type": "module"`) with NodeNext module resolution
- apps/web uses Next.js App Router with bundler module resolution (different from packages)
- @ingot/core is referenced as `"@ingot/core": "*"` in dependent package.json files
- Run `npm run typecheck` from root to check all workspaces
